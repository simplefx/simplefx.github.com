Notes on the Documentation for Simple.Data

Simple.Data offers you the opportunity to code against many databases in more IDEs, so for the purposes of this documentation the following choices have been made

  • IDE : Visual Studio 2010
  • Server : SQL Server 2008
  • Database : MVCMusicStore
  • Language : C#

Sample Code

All samples in this documentation are part of the Simple.Data.Sample package which you can download as a NuGet package or from github at https://github.com/markrendle/Simple.Data.Sample

  • All code snippets in the documentation assume that there is a variable db which was returned from one of the Simple.Data Database.Open...(...) methods.
  • Unless explicitly mentioned, all code snippets which run an operation against the database will also show the SQL generated by that code using the ADO adapter and the SqlServer provider. Obviously Simple.Data is not just a SQL wrapper, but this convention should be most easily understood by the widest audience.
  • In the ‘Generated SQL’ code, parameters will be shown as simple assignments before the SQL in which they are used. This is just a convenient shorthand notation and not valid SQL. At runtime in the ADO adapter, parameters are handled through the IDbCommand.Parameters collection.
